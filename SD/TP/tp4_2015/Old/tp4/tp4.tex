\section{L'algorithme de Dijkstra}

\subsection{Principe}

Il s'applique à des graphes orientés, valués positivement.
Il calcule la longueur minimale des chemins partant de la racine~$A$
vers tous les autres sommets du graphe, accessibles à partir de~$A$.

Chaque sommet est muni d'un {\em potentiel}, qui vaut initialement
$+\infty$, à l'exception du potentiel de~$A$, qui est initialisé à~$0$. 
À la fin de l'exécution de l'algorithme, ce potentiel est égal à la 
longueur minimale recherchée.
L'algorithme de Dijkstra s'implante efficacement avec une file
de priorité. Plus le potentiel d'un sommet est petit, plus il
est prioritaire.
L'algorithme en pseudo-code est donné Figure~\ref{Dijkstra}.
\begin{figure}[h!t]
\begin{tabbing}
\BEGIN \\
\INDENTER
    Mettre le potentiel de $A$ à $0$, tous les autres à $+\infty$ \\
    Enfiler tous les sommets de~$T$ dans la file de priorité $F$ \\
    \WHILE\ $F$ n'est pas vide \DO \\
    \INDENTER
        $X$ = defiler un sommet de $F$ \\
        imprimer ``dist. min. de $A$ à $X$ = le potentiel de $X$'' \\
        Soit $\ell$ l'indice de~$X$ dans~$T$ \\
        \FOR\ $c$ variant de $0$ à $N-1$ \DO\\
        \INDENTER
            \IF\ $B(\ell,c) \neq 0$ \THEN \\
            \INDENTER
            \IF\ $\mbox{\em potentiel}\,(T[c]) > \mbox{\em potentiel}\,(X) + B(\ell,c)$ \THEN \\
            \INDENTER
                $\mbox{\em potentiel}\,(T[c])$ = 
                        $\mbox{\em potentiel}\,(X) + B(\ell,c)$ \\
                reporter un changement de priorité pour $T[c]$ dans $F$ (utiliser
pour cela la \\ \hspace*{2 cm} fonction {\tt changement\_priorite} et le champ {\tt indice} de $T[c]$) \-\\
            \FI \-\\
            \FI \-\\
        \OD \-\\
    \OD \-\\
\END
\end{tabbing}
\caption{\footnotesize L'algorithme de Dijkstra en pseudo-code.
Le graphe est implanté ainsi~:
il y a~$N$ sommets indicés de~$0$ à~$N-1$,
un tableau~$T$ contenant tous les sommets,
une fonction~$B(\ell,c)$ qui retourne la valeur de l'arc de 
$T[\ell]$ vers $T[c]$, ou~$0$ si cet arc n'existe pas.}\label{Dijkstra}
\end{figure}

\subsection{Implantation des graphes}

L'archive {\tt graphe.tgz} (disponible sur le site du cours)
contient les types nécessaires à une implantation minimaliste des graphes.
Le fichier {\tt sommet.h} contient la déclaration du type ci-dessous,
inspiré du type {\tt struct element}. Il permet de représenter un
sommet d'un graphe~:
\begin{verbatim}
struct sommet {
    char nom;
    int potentiel;    /* Pour Dijkstra */
    int indice;       /* Ce champ est utilisé par les files de priorité */
};
\end{verbatim}
Les fichiers {\tt graphe.h} et {\tt graphe.c} contiennent un codage du
graphe de la Figure~\ref{fig-tp4}. Voici les prototypes déclarés
dans {\tt graphe.h}~:
\begin{verbatim}
#include "sommet.h"
/*
 * L'entier N est égal au nombre de sommets du graphe
 * Les sommets sont dans T, (indices de 0 à N-1).
 * La fonction B(l,c) implante la matrice d'adjacence du graphe.
 * Elle retourne le « poids » de l'arc  T[l] -> T[c], 0 si l'arc n'existe pas.
 * La fonction dessin dessine le graphe dans une fenêtre graphique.
 */
extern int N;                              
extern struct sommet T [];                 
extern int indice_dans_T (struct sommet*); 
extern int B (int, int);
extern void dessin (void);
\end{verbatim}

\end{figure}
