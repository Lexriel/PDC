\section{Analyse de l'algorithme de Karatsuba}

On s'intéresse à la méthode de Karatsuba.
On note~$f(n)$ le nombre d'opérations arithmétiques sur les {\tt double}
effectuées.

\paragraph{Étape 1.}
Les deux questions qui suivent sont destinées à établir
la fonction $f_{\mathrm{simplifie}}(n)$, qui donne le nombre
d'opérations sur les {\tt double} par l'algorithme de Karatsuba,
dans le cas simplifié $n = 2^k$.

\question
En analysant le code de la fonction {\tt Karatsuba} de la feuille
de TD, écrire une relation de récurrence avec condition initiale, 
pour la méthode de Karatsuba, dans le cas simplifié où $n = 2^k$.
Faire attention à la condition initiale (multiplication élémentaire
de deux polynômes constants). L'intuition peut être trompeuse. 
Bien regarder le code donné sur la feuille de TD.

\solution{Ça a été fait en TD. On trouve
$$f(n) = 3\,f\left(\frac n 2\right) - 4\,n + 3 \,,\quad f(1)=2\,.$$}

\question
Résoudre cette relation de récurrence avec condition initiale,
en utilisant le logiciel MAPLE (disponible sur le serveur {\tt weppes}).
Dans la suite de cet énoncé, on appellera $f_{\mathrm{simplifie}}(n)$ 
la formule ainsi obtenue. Compléter~:
$$f_{\mathrm{simplifie}} (n) = \hspace*{6cm}$$
Pour vous aider, voici la résolution, avec MAPLE, d'une autre
relation de récurrence avec condition initiale.
\begin{verbatim}
fboulier@weppes$ /usr/local/maple11/bin/maple
> syst := { f(n) = 2*f(n-1) + 3, f(0) = 1 };
                   syst := {f(n) = 2 f(n - 1) + 3, f(0) = 1}

> rsolve (syst, f(n));
                                      n
                                   4 2  - 3
\end{verbatim}
\solution{Voir ci-dessous. On trouve donc
$$f(n) = \frac{17}{2}\,n^{\log_2(3)} - 8\, n + \frac 3 2\cdotp$$}
\begin{quote}
\begin{verbatim}
 syst := { f(n) = 3*f(n/2) + 4*n - 3, f(1)=2};
                 syst := {f(1) = 2, f(n) = 3 f(n/2) + 4 n - 3}

> rsolve (syst, f(n));                         
                                 /ln(3)\
                                 |-----|
                                 \ln(2)/
                           17/2 n        + 3/2 - 8 n

> evalf (%);
                              1.584962501
                 8.500000000 n            + 1.500000000 - 8. n
\end{verbatim}
\end{quote}

une pour~$n$ et une pour~$f(n)$. Aller jusqu'à $n = 600$.

\paragraph{Étape 3.}
Les questions qui suivent sont destinées à se convaincre, au
moins expérimentalement, que le nombre~$f(n)$ d'opérations sur les
{\tt double} effectuées, dans tous les cas, par l'algorithme de Karatsuba,
est bien un équivalent asymptotique de la 
fonction~$f_{\mathrm{simplifie}} (n)$ établie dans le cas simplifié
uniquement. 

\question
Tracer sur un même graphique, avec GNUPLOT, la courbe mesurée
correspondant à l'algorithme de Karatsuba et la courbe théorique du 
cas simplifié~$f_{\mathrm{simplifie}} (n)$ (attention, avec GNUPLOT,
la variable doit être appelée~$x$ et pas~$n$). 
Normalement, les deux courbes doivent se croiser pour tout~$n$ de la 
forme $2^k$. Vérifier. 

\solution{Voici les commandes et les graphiques.}
\begin{quote}
\begin{verbatim} 
log2(x)=log(x)/log(2)
plot "Karatsuba.stats" with lines, 8.5*x**log2(3)-8*x+1.5
\end{verbatim}
\end{quote}

\centerline{\includegraphics[width=.8\textwidth]{courbe3.pdf}}

\question
Où se situe la courbe théorique du cas simplifié~$f_{\mathrm{simplifie}} (n)$ 
par rapport à la courbe mesurée~$f(n)$~? 
En faisant confiance au graphique, 
peut-on affirmer que $f(n) \in \Omega (n^{\log_2(3)})$~? 
que $f(n) \in O (n^{\log_2(3)})$~? 
que $f(n) \in \Theta (n^{\log_2(3)})$~? 

\solution{La courbe est en dessous. Si on fait confiance au graphique,
on peut affirmer que $f(n) \in \Omega (n^{\log_2(3)})$.}

\question
Pour finir de se convaincre que $f(n) \in \Theta (n^{\log_2(3)})$, on
peut appliquer deux méthodes~:
\begin{enumerate}
\item Chercher empiriquement une autre courbe $F (n)$, asymptotiquement
        équivalent à~$f_{\mathrm t} (n)$, située de «~l'autre côté~» de
        la courbe mesurée.
\item Tester la courbe $F (n) \egaldef f(2\,n)$.
        Développer la formule avec MAPLE.
\end{enumerate}

\solution{Si on remplace le coefficient $17/2$ par~$11$, il semble
que la courbe soit au-dessus.}
\begin{quote}
\begin{verbatim} 
log2(x)=log(x)/log(2)
plot "Karatsuba.stats" with lines, 8.5*x**log2(3)-8*x+1.5, 11*x**log2(3)
\end{verbatim}
Avec MAPLE, on trouve bien une courbe de la forme $a\,n^{\log_2(3)}$.
\begin{verbatim} 
> f := 17/2*n^(log[2](3)) - 8*n + 3/2;
                                   /ln(3)\
                                   |-----|
                                   \ln(2)/
                        f := 17/2 n        - 8 n + 3/2

> F := subs (n = 2*n, f);
                                     /ln(3)\
                                     |-----|
                                     \ln(2)/
                      F := 17/2 (2 n)        - 16 n + 3/2

> simplify (%);
                                /ln(3)\
                                |-----|
                                \ln(2)/
                          51/2 n        - 16 n + 3/2
\end{verbatim}
\end{quote}

\paragraph{Étape 4.}
Cette étape apporte quelques compléments aux questions précédentes.

\question
S'il fallait approximer la courbe mesurée par une formule de la
forme $a\,n^{\log_2(3)}$, quelle serait la meilleure valeur pour~$a$~?
Une solution consiste à utiliser la fonction {\tt fit} de GNUPLOT.
Quelle valeur obtient-on~?
$$a = \hspace*{6 cm}$$
\begin{verbatim}
$ gnuplot
gnuplot> log2(x)=log(x)/log(2)
gnuplot> fit a*x**log2(3)+b*x+c "fichier" via a,b,c
\end{verbatim}

\solution{Voici la solution avec gnuplot. On trouve $a \simeq 6.13$}
\begin{quote}
\begin{verbatim} 
log2(x)=log(x)/log(2)
fit a*x**log2(3)+b*x+c "Karatsuba.stats" via a,b,c

Final set of parameters            Asymptotic Standard Error
=======================            ==========================

a               = 6.12836          +/- 0.1542       (2.516%)
b               = 186.537          +/- 9.252        (4.96%)
c               = -17284.9         +/- 1457         (8.431%)
\end{verbatim}
\end{quote}

\question
Dans notre implantation de l'algorithme de Karatsuba, on appelle
la multiplication élémentaire quand~$n = 1$. Or, la multiplication
élémentaire reste meilleure que l'algorithme de Karatsuba pour
des valeurs de~$n$ plus élevées que~$1$.

Modifier en conséquence le seuil à partir duquel l'algorithme de 
Karatsuba devrait appeler la multiplication élémentaire.

Le seuil optimal n'est pas celui qu'on a déterminé graphiquement
quelques questions plus haut. Voyez-vous pourquoi~? Déterminer ce seuil
expérimentalement.

Le nouvel algorithme est-il toujours équivalent asymptotiquement
à~$n^{\log_2(3)}$~?
Si oui, quelle nouvelle valeur donner à la constante~$a$ de
la question précédente~?

\solution{Pas fait.}
