Consignes concernant les travaux informatiques à rendre

Sauf mention du contraire (utilisation de l'outil PROF), les projets et travaux pratiques de contrôles sont remis exclusivement par courrier électronique à l'adresse Alexandre.Sedoglavic@univ-lille1.fr. Le sujet de ce courrier devra être : [ PDC - Remise] designation_du_travail.

Vous devez rendre une archive tar compressée --- d'extension .tgz --- de vos fichiers. Cette archive doit systématiquement contenir :

  • un fichier Readme documentant votre travail (limitation, etc.) et mentionnant vos nom, numéro de groupe et adresse électronique ;
  • des fichiers sources et en aucun cas des fichiers intermédiaires (objets, exécutables ou log). Si le sujet indique des noms pour ces fichiers sources, veillez à respecter scrupuleusement cette consigne. Le cas échéant, vous pouvez fournir des fichiers de test et de documentation (exclusivement sur le modèle du manuel unix) ;
  • si la compilation est nécessaire, un fichier Makefile permettant de reconstruire vos exécutables (et de lancer vos tests si nécessaire).
Attention, le nom du fichier archive sera nécessairement au format nom.tgz ou nom1_nom2.tgz ; nom étant votre nom (nom1 et nom2 pour les binômes).

Consultez le calendrier pour connaître les dates limites de remise.
