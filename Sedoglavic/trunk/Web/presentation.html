Langage C

Objectifs :

L'enseignement est centré sur l'apprentissage du langage C. Ce langage est pensé comme un assembleur portable (l'étudiant intéressé peut consulter le cours d'architecture élémentaire des ordinateurs pour cet aspect).

Par ailleurs, une part importante du module est consacré aux outils de développement (interpréteur de commande, éditeur, make, gdb, etc). Ces derniers permettent l'enchaînement des commandes de compilation, la mise au point des programmes, la gestion de versions, l'utilisation de scripts shell simples, par exemple pour automatiser le test de programmes.

Ce cours vise donc à maîtriser le lien entre l'algorithmique, la programmation, les outils de développement et le matériel informatique. Il ne s'agit pas d'un module de programmation : cet aspect est supposé préalablement acquis.

Remarque :

Les travaux pratiques se font sur ordinateur personnel gérer par le système d'exploitation Linux.

Volume horaire :

Cet enseignement est organisé en une séance de cours-TD et deux séances de TP hebdomadaire (chacune ayant une durée d'une heure et demie). Il est réparti sur une durée de quatorze semaines.

Évaluation :

L'évaluation s'effectue suivant une procédure de contrôle continu, et un examen en fin de semestre.
Trois notes seront attribuées à chaque étudiant durant le semestre :
  • TD : une note sur 20 de travaux dirigés (interrogations écrites, etc.), attribuée par l'enseignant de travaux dirigés ;
  • TP : une note sur 20 de travaux pratiques ;
  • Proj : une note sur 20 sur un projet ;
  • EX : une note sur 20 pour l'examen de fin de semestre.

La note finale sur 20 sera calculée comme une moyenne pondérée de ces trois notes :

N= sup(EX, (2EX+(TD+TP+Proj)/3)/3)

Références bibliographiques :

  • Kernighan, B. W., et Ritchie, D. M. Le langage C. Masson, 1986.
